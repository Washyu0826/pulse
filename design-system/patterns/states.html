Pulse Design System · States

空／錯誤／載入三態

原則：錯誤 ≠ 空狀態（分開文案、分開行為）；文案說使用者的語言，不說「API」；每個狀態都給下一步。插畫式樣＝accent 淡底圓＋單色線性 icon，克制不搶戲。

三態並列

Loading · 載入中

骨架與實際版型同構（避免載入後跳動）；
動畫尊重 prefers-reduced-motion。

Empty · 空狀態

今天還沒有新情報

內容每天早上更新。
可以先回顧昨天的重點，或放寬時間範圍。

Error · 錯誤

資料暫時載入不了

可能是資料來源正在更新。
稍等一下再試，已載入的區塊不受影響。

文案修正對照

使用者語言，開發細節留 console
現況（開發者導向）修正後（使用者語言）
「無法載入情報，請確認 API 是否啟動」「資料暫時載入不了，稍等一下再試」＋〔再試一次〕
API 掛掉與真的沒資料顯示同一句「尚無今日事件」錯誤→可重試錯誤卡；真空→「今天的事件還在聚合中」＋降級顯示昨日事件
材料包失敗直接吐「HTTP 500」「生成失敗了，稍後再試」（429 →「請稍後再試」），狀態碼留 console
次要區塊（摘要列／熱詞）故障時整塊靜默消失降級成淡化 placeholder——同一次故障全站行為一致
