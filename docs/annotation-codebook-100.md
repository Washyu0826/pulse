# Pulse 標註 100 條邊界裁決 Codebook

> 目的：把標註時會卡的邊界情境一次釘死，讓 gold set 一致（提高 κ）。
> 每條 = **情境 → 建議裁決**。建議裁決依使用者已拍板的 J1–J10（見 annotation-guidelines.md §8）
> 與其「實用情報」品味（重資訊傳遞 > 原創；行銷/洗版/純作品壓低）預填。
> **⚠ = 真正有爭議、請重點確認**；其餘多為前述原則的延伸。建立 2026-06-08。
>
> 標籤：情緒 {positive, neutral, negative}｜品質 {high, mid, low}｜
> 主題 {新工具, 模型動態, 使用方法, 風險限制, 倫理法規, 其他}

---

## A. 情緒 Sentiment（作者對其談論的 AI 工具/模型/做法的態度）

1. 新工具/新版**純公告**、作者無表態 → **依語氣**（興奮/推薦→positive；純陳述→neutral）。〔J1〕
2. 比較貼**純陳述**差異（A 比 B 快）→ **neutral**；帶推薦傾向 → positive。
3. 在抱怨、但**抱怨對象不是 AI** → **neutral**。〔J7〕
4. 反諷（表面褒、實貶）→ 標**真實意圖**（多為 negative）。
5. 又愛又恨、正負強度相當 → **neutral**（備註 mixed）。
6. 提問/求助（「有人知道怎麼解 X 嗎」）→ **neutral**。
7. 台灣俚語「香」「神器」「好用到哭」「太頂」→ **positive**。
8. 「很雷」「踩雷」「翻車」「智障」→ **negative**。
9. 中英混罵（reasoning 有夠扯、根本沒在 think）→ **negative**。
10. 純功能教學、全程無評價 → **neutral**。
11. 推薦但附「但要注意 X」→ 看主導，多為 **positive**。
12. ⚠ 客觀 bug/缺陷回報：**指出嚴重缺陷 → negative；中性回報、無批評語氣 → neutral**。
13. 「AI 取代工作」焦慮 → **一律 negative**（視為對 AI 的負面情緒）。〔拍板〕
14. 某模型降價/開源「太佛了」「佛心」→ **positive**。
15. 「越來越爛」「退步了」「變笨」對某模型 → **negative**。
16. 純轉貼新聞、無評論 → **neutral**（品質另計，見 61）。
17. ⚠ emoji-only / 無文字回應 → **neutral 且建議標低信心**（資訊不足）。
18. 迷因/梗圖（少文字）→ **可辨識意圖就依意圖；不可辨 → neutral**。
19. 「跪求」「拜託出 X 功能」許願 → **neutral**。
20. 比較後選邊「我還是用 X」→ 對**主體 X 標 positive**（不替被比下去的標負）。
21. 「付費值得嗎？」討論 → **neutral**（提問）。
22. 炫耀自己用 AI 的成果（隱含對 AI 正面）→ **positive**。
23. ⚠ 對 AI 危害的擔憂（隱私/造假/倫理）→ **陳述風險、無批評語氣 → neutral；明確譴責 → negative**。
24. 政治/社會爭議中提到 AI → **只標對 AI 的態度，忽略政治情緒**。
25. 「請大家小心 X 詐騙」警告 → **negative**（警告勸退）。

## B. 主題 Theme（這篇主要在講什麼；平手取篇幅最多者，J10）

26. 比較/跑分/排名/價格「A vs B 誰強」→ **模型動態**。〔J2〕
27. 玩 AI / 考倒 AI / @meta.ai 互動洗版 → **其他**。〔J5〕
28. 敲碗/許願：有具體情境 → **使用方法**；純許願無內容 → **其他**。〔J8〕
29. 提示詞模板 / 咒語大全 → **使用方法**。
30. 新工具發表 **+ 附用法** → 看主導（重點在用法→使用方法；在發表→新工具）。
31. 模型新版發表純宣布 → **新工具**（若強調跑分/能力提升 → 模型動態）。
32. AI 繪圖**作品分享**（無方法）→ **其他**。
33. AI 繪圖**教學**（附提示詞/流程）→ **使用方法**。
34. 「AI 會幻覺、要查證」實務提醒 → **風險限制**。
35. 詐騙/deepfake/造假 → **一律風險限制**（即使是社會危害面；倫理法規保留給職涯/監管/隱私）。〔拍板〕
36. AI 隱私 / 資料外洩 / 「別把 X 餵給 AI」→ **倫理法規**。
37. AI 版權 / 訓練資料爭議 → **倫理法規**。
38. AI 監管 / 法案（EU AI Act、各國規範）→ **倫理法規**。
39. AI 取代工作 / 職涯衝擊：**社會衝擊面 → 倫理法規；「如何用 AI 強化職涯」的方法 → 使用方法**。〔拍板〕
40. agent / MCP / 自動化**工具發表** → **新工具**。
41. agent / 自動化**工作流教學** → **使用方法**。
42. 模型**開源釋出權重** → **新工具**（若重點在跑分 → 模型動態）。
43. 工具/模型「哪個 best for X」推薦比較 → **模型動態**。
44. ⚠ 純情緒抱怨某工具爛、無比較無方法 → **風險限制**（指向缺陷/限制）。
45. 單一工具實測/心得 → **使用方法**（use case）。
46. 公司融資/收購/商業動態 → **其他**（非 5 實用主題）。〔拍板〕
47. 研究突破/論文分享 → **其他**（純研究；若明確是模型能力更新 → 模型動態）。〔拍板〕
48. AI 硬體 / GPU / 算力 → **其他**。
49. 課程/活動/社群宣傳（非帶課變現）→ **其他**。
50. 「AI 新聞懶人包」彙整多則 → 看主導主題；跨太多 → **其他**。
51. AI 音樂/影片/語音工具發表 → **新工具**。
52. 「我做了一個 AI 小工具」自製分享 → **新工具**（附教學 → 使用方法）。
53. prompt 比賽/挑戰 → **使用方法**（若純玩樂洗版 → 其他）。
54. AI 與教育/學習方法 → **使用方法**。
55. AI 幻覺/錯誤的**技術原理**解釋 → **風險限制**。
56. 「AI 會不會有意識/AGI」哲學討論 → **其他**。
57. 模型 benchmark 榜單更新（Arena/MMLU）→ **模型動態**。
58. 工具**停服/漲價/政策變更** → **模型動態**（價格/服務變動）。
59. 個資/未成年/AI 陪伴的倫理 → **倫理法規**。
60. 完全與 AI 無關（誤抓）→ **其他**（且品質 low）。

## C. 品質 Quality（對「每日 AI 實用情報」的價值）

61. 純轉貼新聞連結、無評論 → **high**（重資訊傳遞 > 原創）。〔J6〕
62. 變現/帶課「教你用 AI 月入 X 萬」→ **low**（行銷雜訊）。〔J3〕
63. AI 繪圖純作品分享（圖+一句）→ **low**。〔J9〕
64. 玩 AI / 洗版互動 → **low**。
65. 具體可複用的教學（步驟/提示詞/實測數據）→ **high**。
66. 一句技巧、無展開（「叫它一步步想」）→ **low**（資訊量太低；與「低資訊一律壓低」一致）。〔拍板〕
67. 深度長文教學/拆解 → **high**。
68. 泛泛而談、無乾貨（「AI 真的很厲害」）→ **mid**。
69. 廣告/聯盟行銷/「點我連結」→ **low**。
70. SEO 農場/帶風向/標題殺 → **low**。
71. 機器人/疑似自動發文 → **low**。
72. 重要官方發表的轉述（即使無評論）→ **high**。
73. 求助提問（具體技術問題）→ ⚠ **mid**（對讀者價值中等；社群有討論才升值）。
74. 純抱怨情緒宣洩、無資訊 → **low**。
75. 實測比較含數據/截圖 → **high**。
76. 個人日常用 AI 小記（輕、但具體 use case）→ **mid**。
77. 迷因/梗圖娛樂 → **low**。
78. 重複/洗稿/搬運他人內容 → **low**。
79. 工具清單/懶人包（有整理價值）→ **high**（純羅列無說明 → mid）。
80. 提示詞模板大全（可直接複用）→ **high**。
81. 「敲碗許願」無內容 → **low**。
82. 有爭議但有料的觀點（引發討論）→ **high**。
83. 過時/舊聞（資訊已不新）→ **mid**（仍相關但價值降）。
84. 招募/徵才/接案貼 → **low**（非情報）。
85. 安全/隱私/邊界的具體提醒（可行動）→ **high**。

## D. 範圍 / 收錄 Scope

86. 主要英文、夾少量中文（CJK ≥ 20%）→ **收進 gold**。〔J4〕
87. CJK < 20%（幾乎全英）→ **不收**（中文 gold 純化）。
88. ⚠ 簡體中文（中國內容）→ **不收**（爬蟲已過濾；漏網的標註時剔除）。
89. 純圖片貼、無可讀文字 → **不收**（無文字可標）。
90. 文字 < 20 字、資訊不足 → **可收但標低品質**；< 5 字 → 不收。
91. 廣告/spam（DQC 已標）→ **不收進 gold**（除非要做反例集）。
92. 同一作者短時間洗多篇雷同 → **只收一篇代表**（去重）。
93. 轉發他人貼（quote/repost）→ **收**，以被轉內容為主。
94. 含敏感/NSFW 但與 AI 相關 → **收**，照常標（不做語言/內容審查）。
95. 台語/方言夾雜的 AI 貼 → **收**（屬台灣在地訊號）。

## E. 格式 / 特殊 Special

96. 一篇多句、情緒不一致 → 標**整體主導**情緒；難判 → neutral + 備註。
97. 標題與內文主題不同 → 以**內文**為準。
98. 內含程式碼區塊的技術貼 → 照常標（程式碼不影響情緒）。
99. 串文（thread）只抓到其中一則 → 就**該則可判定的**標；脈絡不足 → neutral + 備註「脈絡不足」。
100. 不確定/真的難判 → **情緒 neutral、主題 其他、品質 mid**，**務必寫備註**，事後集中複審。

## 怎麼用這份

- 標註時遇到卡點，先在這查；查不到的新型態 → 標完備註、回來補一條。
- 這 100 條是 [[annotation-guidelines]] §8（J1–J10）的擴充；衝突時以你後續裁決為準。
- **請你重點看 ⚠ 的 13 條**（12, 13, 17, 23, 35, 39, 44, 46, 47, 66, 73, 88 …）—— 那些是我替你預設、但你可能有不同想法的。
