# Pulse 標註指南（Annotation Guidelines / Codebook）

> 目的：建立**中文為主**的 gold set，供模型選型的 Offline Evaluation、DQC 門檻驗證、
> 溫度校準使用（落地 ADR-008）。本指南是標註的單一事實來源；標註難例先翻這份。
>
> 工具：`python scripts/annotate.py`（純函式骨架在 `ml/ml/annotation.py`）。
> 版本：v1（2026-06-05）。修訂指南時務必升版號並重標受影響樣本。

---

## 1. 為什麼要這份指南（方法學依據）

主觀任務（情緒、立場）的標註一致性，取決於**事先寫清楚的判準**，而不是標註當下的直覺。
本指南的設計參考：

- **C-STANCE (ACL 2023)** / **ChnSentiCorp**：中文社群情緒/立場標註的標籤集與慣例對齊。
- **「Just Put a Human in the Loop?」(ACL 2025)**：警告 LLM 預標會讓人錨定 → 人工標註時**先不要看模型答案**。
- **IAA 指南 (2026)**：一致性要報**信賴區間**（bootstrap CI），不要只報裸 κ。
- **Confident Learning / cleanlab (JAIR 2021)**：完工後用模型 out-of-fold 機率自動揪出可疑標籤再複審。

---

## 2. 標註單位與抽樣

- **單位**：一篇貼文（標題 + 內文），不標留言。
- **抽樣**：`scripts/annotate.py` 用 `stratified_sample` 依 `source` 分層、固定 seed 隨機抽，**不得 cherry-pick**。
- **中文優先**：加 `--zh` 只抽 CJK 字元比 ≥ 20% 的貼文（Threads 為主要中文來源）。
- **目標量**：gold set 首批 **200–500 筆**；其中重標 **20 筆**做 self-consistency。

---

## 3. 三個標註維度

每篇同時標三項：**情緒**、**品質**、**主題**。
`annotate.py` 互動標這三項（情緒 1/2/3 → 品質 h/m/l → 主題 t/m/u/r/e/o）。
舊 gold（無 theme 欄位）仍可用；重跑 `annotate.py` 會補標新貼文的主題。

### 3.1 情緒 Sentiment（按鍵 1/2/3）

判準：**貼文作者對其談論的 AI 工具/模型/做法所表達的態度**（不是貼文讓你心情如何）。

| 標籤 | 鍵 | 判準 | 範例 |
|---|---|---|---|
| `negative` 負 | `1` | 抱怨、失望、批評、警告勸退、指出嚴重缺陷 | 「DeepSeek 一直 timeout，退回 Claude 了」 |
| `neutral` 中 | `2` | 純資訊/提問/中立陳述，無明顯褒貶；或褒貶相抵 | 「Claude Skills 發表了，提供打包 agent 能力的新方式」 |
| `positive` 正 | `3` | 推薦、稱讚、驚艷、表達偏好/滿意 | 「Qwen2.5 本地跑翻譯品質意外地好」 |

**邊界規則：**
- **新工具發表公告**本身 → `neutral`，除非作者明顯表達興奮/推薦。
- **客觀比較**（A 比 B 快）→ 看作者立場：純陳述 = `neutral`；帶推薦 = `positive`。
- **提問/求助**（「有人知道怎麼解決 X 嗎」）→ `neutral`。

### 3.2 品質 Quality（按鍵 h/m/l）

判準：這篇對「**每日 AI 實用情報**」的價值（對齊 ADR-009 DQC 帶）。

| 標籤 | 鍵 | 判準 |
|---|---|---|
| `high` 高 | `h` | 具體、可行動、有資訊量（新工具細節、可複用的用法、實測心得） |
| `mid` 中 | `m` | 相關但普通（泛泛而談、舊聞、低資訊量但不算雜訊） |
| `low` 低 | `l` | 雜訊：廣告/SEO/帶風向、無關、純情緒宣洩、標題殺、機器人文 |

### 3.3 主題 Theme（按鍵 t/m/u/r/e/o）

判準：這篇**主要在講什麼**（對齊 theme.py 5 主題 + fallback；2026-06 依 AI 資訊需求研究改版，
把原「邊界」拆成 風險限制／倫理法規，並新增 模型動態）。標**主導**主題；若兼涉多個，取篇幅/重點最多者
（模型分類器會另外輸出 top-2，人工只標主主題即可）。

| 標籤 | 鍵 | 判準 | 範例 |
|---|---|---|---|
| `新工具` | `t` | 新 AI 工具/app/產品/功能**發表或介紹** | 「Anthropic 推出 Claude Skills」 |
| `模型動態` | `m` | 模型**比較/評測/排名/價格/能力更新**（哪個最好） | 「GPT-5 vs Claude 在 SWE-bench 誰強、順便比價」 |
| `使用方法` | `u` | 提示技巧/教學/工作流/use case/實作步驟（**錨點主題**） | 「分享我的 prompt 工作流：先列大綱再展開」 |
| `風險限制` | `r` | **實務**限制/失敗/幻覺/不該用、要查證 | 「Claude 會編造 API，重要的事要自己查證」 |
| `倫理法規` | `e` | 倫理/法規/政策/**隱私**/版權/監管 | 「歐盟 AI Act 上路，高風險用途要注意資料隱私」 |
| `其他` | `o` | 以上皆非、閒聊、純情緒、無實質 AI 內容 | 「今天天氣真好」 |

**邊界規則（最容易混的幾組）：**
- **新工具 vs 模型動態**：只是「有這個新東西」→ `新工具`；重點在「A 比 B 好/跑分/多少錢」→ `模型動態`。
- **新工具 vs 使用方法**：「發表 + 怎麼用」混合 → 看主導，重點在用法標 `使用方法`，重點在發表標 `新工具`。
- **風險限制 vs 倫理法規**：**個人實務**踩雷（幻覺、越權、要查證）→ `風險限制`；**社會/政策/法律/隱私**層面 → `倫理法規`。
- **抱怨某工具有缺陷/幻覺** → `風險限制`（即使提到工具名）。
- 與情緒**正交**：一篇可以是「`新工具` + `negative`」（發表了但作者不看好）。

## 4. 中文標註難例（最重要的一節）

### 4.1 反諷 Sarcasm
中文反諷常無 `/s` 標記，靠語境。表面褒、實貶 → 標**真實意圖**的情緒。
> 「喔真棒，又是一個只會 demo 的 AI 工具 👏」→ `negative`（不是 positive）。

### 4.2 中英混雜 Code-switching（Threads 常見）
- 不因語言切換改變判準；看整體語意。
- 專有名詞（Claude / RAG / GPT-4）保留原文不影響情緒判斷。
> 「這個 agent 的 reasoning 真的有夠扯，根本沒在 think」→ `negative`。

### 4.3 繁簡與台灣用語
- 以**台灣繁中**語感判讀（「超讚」「很雷」「踩雷」「神器」）。
- 「很雷」「踩雷」= `negative`；「神器」「香」= `positive`。

### 4.4 回文脈絡缺失
- 只標貼文本身可判定的情緒；若是回覆某串、脫離脈絡無法判 → 偏 `neutral`，並在備註寫 `脈絡不足`。

### 4.5 又愛又恨（混合情緒）
- 正負並陳且強度相當 → `neutral`，備註 `mixed`。
- 一面倒則取主導面。
> 「便宜歸便宜，但常壞掉」→ 主導是抱怨 → `negative`。

### 4.6 模型/工具名歧義
- 「grok」「llama」「gemini」等詞在非 AI 語境（占星、寵物）出現 → 該篇多半 `low` 品質、情緒 `neutral`。

## 5. 一致性目標（IAA）

- **self-consistency**：隔 ≥ 2 天盲標重複 20 筆（`--relabel 20`），算 **Cohen's κ**。
- **目標**：κ **> 0.8 良好**、**≥ 0.67 可接受**（低於則修指南、補難例後重標）。
- **報告方式**：κ 必須附 **bootstrap 95% CI**（`annotation.py` 的 `bootstrap_ci`）。
- 多標註者情境（未來找同學交叉標）改報 **Krippendorff α**（`krippendorff_alpha`，支援多人/缺值）。

---

## 6. 流程 SOP

1. `python scripts/annotate.py --target 200 --zh` → 一次標一批，可隨時 `q` 存檔離開、之後續標。
2. **先不要看模型預測**（避免錨定偏差）。
3. 不確定的標 `neutral` 並寫備註，事後集中複審。
4. 標滿後隔幾天 `python scripts/annotate.py --relabel 20` → 看 κ 是否達標。
5. （Phase 3 後）訓練模型 → 用 cleanlab 在 gold set 上揪可疑標籤 → 複審 → 定版 `gold_zh_v1`。
6. gold set（`data/gold/*.jsonl`）**納入版控**，是專案資產。

## 8. 判斷裁決（gold set owner rulings，逐題拍板）

標註時最常卡的邊界，由 gold set 擁有者逐題拍板，確保一致（2026-06-07 起）：

- **J1 發表公告的情緒**：作者沒明顯表態的「新工具/新版發表」純公告 → **依內文語氣逐篇判**
  （有興奮/推薦語氣 → positive；純事實陳述 → neutral）。不套死「一律 neutral」。
- **J2 比較/跑分貼的主題**：「A vs B 誰強 / 跑分 / 排名 / 價格」→ **模型動態**（不是新工具、不是使用方法）。
- **J3 變現/帶課貼的品質**：「教你用 AI 月入 X 萬」這類變現/帶課貼 → **一律 low**（視為行銷雜訊）。
- **J4 中英混雜的收錄**：主要英文、只夾少量中文的 AI 貼 → **收進 gold**（CJK 字元比 ≥ 20% 就算，反映 Threads 中英混排現實）。
- **J5 玩 AI / 洗版貼的主題**：「玩 AI / 考倒 AI / @meta.ai 互動」無實質情報的洗版貼 → **其他**。
- **J6 純轉貼新聞的品質**：純轉貼 AI 新聞連結、自己沒評論 → **high**（重視資訊傳遞價值 > 原創性；連結內容是重要情報）。
- **J7 抱怨對象非 AI 的情緒**：貼文在抱怨但對象不是 AI（只順帶提到用 AI）→ **neutral**（情緒只看「對 AI 的態度」）。
- **J8 敲碗/許願貼的主題**：「有沒有 AI 可以幫我做 X？」→ **有具體情境的需求 → 使用方法（use case）；純許願無實質內容 → 其他**。
- **J9 AI 繪圖作品分享的品質**：只貼圖 + 一句、無方法/無工具名 → **low**（純作品展示、無可複用資訊）。
- **J10 多主題貼的平手取捨**：一篇跨多主題、主主題平手時 → **取篇幅/詳細度最多者**（模型另以 top-2 補次主題）。
