# ADR-006：不做使用者登入

**狀態**：Accepted
**日期**：2026-05-08

## 背景

是否要做 Google OAuth / 帳號系統？

## 選項

1. **完全不做**：公開頁面，所有人看一樣
2. **做 Google OAuth**：個人化訂閱、查詢歷史

## 決定

選 **完全不做**（v1）。

## 理由

1. **MVP 求快**：登入會多 1-2 週工作量
2. **Pulse 個人版**：自己會用就好，不是 SaaS
3. **公開 API 反而是優勢**：別人可以直接用
4. **自訂查詢用 URL 帶參數**：可分享連結

## 後果

**好處**
- 開發快
- 無使用者資料管理負擔
- API 可被外部使用

**代價**
- 無法做訂閱、書籤、查詢歷史等個人化
- 無法限速到「每使用者」級別
- 無法做付費功能

## 暫時的替代方案

- **訂閱週報**：用 email 表單收集，不做帳號
- **個人偏好**：暫時用 URL 參數 / localStorage

## 之後可能要重新考慮的情境

- 想做付費版
- 想統計每個使用者的查詢模式
- 有人問「能不能加我自己的關鍵字訂閱？」
