# Pulse 口頭報告逐字講稿（約 7 分鐘）

## 開場（約 30 秒）｜封面頁

大家好，我的題目是 **Pulse**。
一句話來說，它是一個「**AI 模型輿情情報儀表板**」——
自動從多個技術社群，蒐集大家對主流 AI 模型的真實討論，
然後轉換成可以做決策的訊號：這個模型最近**口碑好不好**、有沒有**出包或重大發布**、選型時**該選哪一個**。
我監測六個模型：GPT、Claude、Gemini、Grok、Llama、DeepSeek。

---

## 1. 動機（約 1 分半）｜動機頁

[換頁]
先講為什麼要做這個。

AI 模型現在迭代非常快，幾乎每週都有新版本、新模型、或定價跟服務的變動。
對工程師來說，**選型和追蹤的成本很高**。

那現有的管道其實都有缺口：
官方的 changelog 只會報喜，看不到真實踩雷；
**HackerNews 或社群**有真實討論，但**只有讚數跟留言，要一篇一篇人工看**，沒有模型維度、沒有情緒、也沒有彙整。

所以真正的痛點是——**沒有一個地方，把「跨社群的真實使用評論」變成「可以決策的訊號」**。

[指向對照表] 這也是我跟 **HackerNews 最大的差異**：
HN 只有讚數，要你自己讀；
Pulse 幫你**以模型為單位彙整**，多了**情緒口碑、自動事件偵測、跨來源去重、還有選型建議**。

---

## 2. 規格（約 1 分鐘）｜規格頁

[換頁]
功能上，核心有幾塊：

第一，**模型看板**，六個模型的即時彙總：討論量、近期趨勢、最新發布、跟口碑指數。
第二，**事件偵測**，自動抓三種值得注意的事：**討論突增、重大發布、跟口碑翻轉**。
第三，**情緒口碑**，用紅綠燈呈現每個模型的風評。
第四，**決策報告**，像是「做 coding 該選 Claude 還是 GPT」，用真實數據給有證據的建議。
另外還有**資料品質過濾**跟**自動排程告警**。

技術上是 FastAPI 加 PostgreSQL 當後端，Apache Airflow 做排程，
情緒分析用 RoBERTa 深度學習模型，前端是 Next.js。

---

## 3. 架構（約 1 分半）｜架構頁

[換頁]
架構的核心是**一條資料流**：
[指流程圖] 多來源爬蟲 → 原始貼文 → 品質過濾跟去重 → 情緒分析 → 事件偵測 → 最後到 API 跟前端。

資料來源這邊有個重要的設計決定。
我原本規劃用 Reddit，但拿不到 API key；後來發現 X 也沒有免費的官方 API。
所以我改採「**多個免 key 的來源**」策略——
HackerNews、Dev.to、Lobsters、Hugging Face、GitHub，這些都不用金鑰。
**反而更穩、成本是零、也分散了風險。**

整條線用 **Airflow 排程**，我寫了七個 DAG，每十五分鐘自動爬，
而且用「**資料感知排程**」——有新資料才觸發下游偵測，不會空轉。

這裡我想特別點一個工程上的決定：
Airflow 限定只能用舊版的 SQLAlchemy，但我的業務程式用新版，兩個沒辦法共存。
我用「**雙環境隔離**」解掉——讓 Airflow 跟業務程式各跑在自己的環境裡。
這其實是實務上 MLOps 很常遇到的依賴衝突問題。

## 4. 目前進度（約 1 分半）｜進度頁

[換頁]
進度的部分，**核心 pipeline 已經整條跑通，而且是真實資料**，我列幾個數字：

爬進來大概 **五千一百多篇貼文**，涵蓋今年四到五月，加上將近兩百筆模型發布事件。

品質過濾這塊，每篇打分數加標記，還做跨來源去重——
結果是高品質五千多篇，並且找出**兩百多組、三百多篇重複**。

情緒分析是用 GPU 跑 RoBERTa，算出每個模型的口碑，
例如 DeepSeek 口碑最高；口碑翻轉我還加了**統計檢定**，確定不是雜訊才報。

事件偵測目前抓到二十幾筆討論突增、九十幾筆發布。

前端剛做完一輪**可用性大升級**：現在新訪客一眼看得懂怎麼用，
還可以點進每個模型，看它的**討論量跟口碑趨勢圖**。

工程品質方面，有**一百五十多個自動化測試、CI 綠燈**，
而且每個階段都「研究 → 實作 → 獨立 code review → 記錄」。

## 5. 遇到難點（約 1 分半）｜難點頁

[換頁]
最後講幾個比較有挑戰的點。

第一個是剛剛提到的**依賴版本衝突**，用環境隔離解掉。

第二個是**資料品質的假陽性**：
品質過濾剛上線時，有六百多篇正常貼文被誤判成垃圾。
我追下去發現，是 HackerNews 內文的 HTML 編碼，被我的規則誤當成 hashtag。
修掉之後，誤判從幾百篇降到只剩三篇。
這讓我體會到——**規則型的過濾，魔鬼都在資料的細節裡**。

第三個是**網路環境**：我所在的網路有 TLS 憑證攔截，
導致 pip、npm、爬蟲、連 Docker build 全部憑證驗證失敗，這些都各自找到解法。

這些坑我都有寫成一份「**開發問題全紀錄**」，症狀、根因、解法都留著。

## 附：可能被問到的問題（先想好答案）

- **Q：你的方法創新/難度在哪？**
  A：三點——把雜亂討論轉成**結構化決策訊號**（不只是爬蟲）；用**穩健統計＋深度學習情緒**做事件與口碑偵測；以及一套**完整的資料工程管線＋MLOps 排程**（多源、品質、去重、自動化、可觀測）。

- **Q：情緒模型準嗎？**
  A：用的是業界常用的 RoBERTa 情緒模型；我有加溫度校準跟信心門檻，低信心的會標記起來；口碑是**信心加權＋小樣本收縮**，避免少量樣本給極端值。

- **Q：為什麼不用 Reddit / 官方 API？**
  A：Reddit 拿不到 key、X 沒有免費 API。改用多個免 key 來源反而更穩、零成本、風險分散；X 跟 Threads 做成選配。

- **Q：資料量會不會太少？**
  A：目前五千多篇是回填四到五月的；系統是**持續即時累積**的，量會自然增加。

- **Q：跟現成工具差在哪？**
  A：現成社群監聽工具多半是泛用、針對品牌；Pulse 是**專門針對 AI 模型**，有模型維度、發布偵測、跟選型決策，這些是通用工具沒有的。
