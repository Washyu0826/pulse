# Pulse UX 設計與驗收 — 定位 C（每日實用情報為面、模型當篩選）

> 定位拍板見記憶 `product-positioning.md`（2026-05-31）。
> 門面 = 每日實用情報，以 3 主題為主軸（🆕 新工具 / 🛠️ 使用方法 / 🚧 邊界）；
> 模型 + 情緒 + 來源 = 篩選維度。本文用**真實 DB 資料**設計畫面並附**可驗收標準**。

---

## 1. 使用者與每日要做的事（JTBD）

**主要使用者**：每天想跟上 AI 圈、但沒時間翻 HN/Reddit/Threads 的工程師 / 技術工作者（也就是你自己）。

| Job（情境） | 頻率 | 對應畫面 |
|------|------|---------|
| 「今天 AI 圈有什麼**新工具**值得看？」 | 每天 | 首頁 · 新工具區 |
| 「有沒有**更好的用法 / 提示技巧**？」 | 每天 | 首頁 · 使用方法區 |
| 「有什麼**坑 / 限制 / 要注意**的？」 | 每天 | 首頁 · 邊界區 |
| 「我只在意 **Claude**，其他別吵我」 | 看心情 | 模型 filter |
| 「過去一週 **Claude 的新工具**整理給我」 | 偶爾 | 查詢頁（主題+模型+時間） |

**設計原則**：首頁要在 **5 分鐘內**讓人掃完今天三類重點；任何深入都靠 filter 收斂，不靠翻頁。

- Threads 量小但中文在地 → 用 `🌏 中文` 標記凸顯差異化，不要被英文淹沒。


**卡片內容**：標題/摘要、模型 badge（可多個）、來源 badge、情緒點（🟢/🟡/🔴/⚪未分析）、相對時間。
Threads 卡加 `🌏` 凸顯中文在地。

**互動**：
- 點模型 chip → 三個主題區**同時**只剩該模型的內容（單頁、不換頁）。
- 點情緒/來源/時間 → 同步收斂三區。
- 點「看全部」→ 進主題列表頁，帶著當前 filter。
- 點卡片 → 單篇頁（原文連結 + 模型/主題/情緒 meta）。

### 4.2 主題列表頁 `/theme/新工具?model=claude&days=7`

同 filter 列 + 該主題的完整列表（排序：最新 / 最高互動 / 最高信心可切換），無限捲動或分頁。

### 4.3 查詢頁 `/query`

主題 ✓ + 模型 ✓ + 時間 ✓ + 情緒 ✓ → 結果列表。可「存成今日精選」/ 匯出。
（這是舊「自訂查詢」重定義：以主題為第一級條件。）

### 4.4 模型頁 `/models/[slug]`（保留、重定位）

不再以「口碑指數」為主角，改成「**Claude 的主題切片**」：該模型的 新工具/使用方法/邊界 各幾則 + 列表，
口碑指數退成頁內一個小區塊。

## 5. 空狀態 / 邊界情況

| 情況 | 處理 |
|------|------|
| 某主題今日 0 則 | 顯示「今天這類沒有新內容，看看近 7 天 →」不留白 |
| 某模型 filter 下三區都空 | 「Claude 近 24h 沒有實用情報，試試放寬時間」 |
| 貼文未分析情緒（Threads 新進） | 情緒點顯示 ⚪（未分析），不假裝中性 |
| 主題低信心（其他） | 不出現在三區；只在「全部/其他」檢視 |
| 內容含相對時間雜訊（Threads 標題夾「2天」） | 顯示前清掉前綴（作者/時間）再呈現 |

## 6. 驗收標準（Acceptance Criteria）

逐項可勾、可測。分「定位一致性」與「功能正確性」。

### A. 定位一致性（最重要 — 確認沒走回舊定位）
- [ ] **A1** 首頁第一屏主角是 3 主題的內容卡，**不是**模型口碑看板。
- [ ] **A2** 模型、情緒、來源在首頁都是**篩選器**（chip/toggle），不是主內容區塊。
- [ ] **A3** 「選型決策報告 / 口碑指數」**不在**首頁第一屏（收合或次要頁）。
- [ ] **A4** 「其他」主題預設**不顯示**於首頁三區。
- [ ] **A5** Threads（中文在地）內容有 `🌏` 標記且能在首頁出現，不被英文淹沒。

### B. 功能正確性
- [ ] **B1** 點任一模型 chip，三個主題區同時只剩該模型內容（單頁更新、URL 帶 `?model=`）。
- [ ] **B2** 情緒 / 來源 / 時間 filter 同步收斂三區，且可組合（如 Claude + 🟢 + 近7天）。
- [ ] **B3** 每張卡正確顯示：模型 badge、來源、情緒點、相對時間；多模型貼文顯示多個 badge。
- [ ] **B4** 「看全部」進主題列表頁時**保留**當前 filter。
- [ ] **B5** 今日摘要列的三主題計數 = 該 filter 下各主題實際卡數。
- [ ] **B6** 未分析情緒的貼文顯示 ⚪（不顯示為中性 🟡）。
- [ ] **B7** 空狀態文案正確（見 §5），不出現空白區塊。

### C. 資料/效能
- [ ] **C1** 首頁只查「高信心、達品質門檻、近 N 天」的貼文（不撈全表）。
- [ ] **C2** 三主題卡片各取 top N（預設每區 4–6 張），有索引支撐（themes.label / quality_score）。
- [ ] **C3** 首頁 API 單次回應 < 500ms（本機 docker）。

### D. 可用性（人工驗）
- [ ] **D1** 找一位真實使用者（你自己）連續 3 天每天開首頁，記錄「5 分鐘內是否掃完三類重點」。
- [ ] **D2** 隨機抽 20 張卡，人工判斷主題分類是否合理（目標 ≥ 80% 認同）。
- [ ] **D3** 抽查 Threads 中文卡，確認標題清乾淨（無「2天/作者名」雜訊前綴）。

## 7. 落地順序（對應後端/前端工作）

1. **API**：`GET /api/feed?model=&sentiment=&source=&days=&theme=` → 回各主題 top N（join themes+sentiments+posts+models）。
2. **API**：`GET /api/feed/summary` → 今日三主題計數。
3. **前端**：首頁改三主題分區 + 共用 filter 列（取代現有 ModelBoard 當主角）。
4. **前端**：主題列表頁 + 查詢頁以主題為第一級。
5. **資料**：Threads 標題清洗（去相對時間/作者前綴）。
6. **文件**：README 差異化表改寫（vs 電子報，而非 vs HN 的口碑/選型）。
